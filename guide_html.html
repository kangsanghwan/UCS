HTML Guide

기본설정

HTML5

HTML5는 기존의 HTML와 동일한 마크업 언어 입니다. 따라서 기본 골격은 기존 HTML과 다르지 않으며 HTML5는 XHTML에서 발전된 버전 입니다.
최신HTML로써 추가 플러그인과 엑티브엑스 없이 멀티미디어 즉 동영상과 오디오플레이를 쉽게 볼수 있다는 장점을 지니고 있고, html, css, javascript를 조합하여 대화형 웹사이트 제작 할 수 있습니다.
HTML4에 기반하고 네임스페이스나 스키마가 없기 때문에 요소(Element)의 끝을 정확히 닫지 않거나 따옴표를 생략해도 괜찮습니다.
HTML5를 지원하지 않는 브라우저에서도 컨텐츠가 무리없이 출력되도록 설계되었으며, 호환성이 고려된 추가적인 새 요소들이 대거 추가되었습니다.
HTML5는 HTML4와 XHTML1 문법과 호환되지만, HTML4 일부 SGML(Standard Generalized Markup Language) 문법과는 호환되지 않는 특성을 가지고 있다.

XHTML 1.0

웹페이지를 제작하기 위해 사용되는 HTML4을 XML에 맞도록 재정의한 언어 입니다. html과 다른 점은 html보다 조금더 구조화된 형식과 엄격한 문법을 가지고 있습니다.
예를 들어 모든 요소는 반드시 시작과 함께 종료를 표현해야함, 소문자사용, 속성값의 생략 불가능,모든 속성값에 인용부호, 특수문자 변환사용,
사용또한 xhtml은 html과 다르게 HTML 4.01의 후속 버전이 아닌 별개로 분리된 xml의 응용 버전이라 볼 수 있습니다.
장점은 html보다 웹사이트 검색기능 향상과 복잡한 데이터 처리에 보다 쉽다는 장점을 가지고 있고, html의 단점을 보안하기 위해 만들어진 언어 입니다.

종료 태그

img태그나 input 태그 등 종료태그가 없는 태그들을 HTML5에서는 /가 없는 것을 허용하였으나,
XHTML과 HTML 일반은 반드시 종료태그를 갖습니다.

<input /> <img /> <br />

모든 이미지 alt 속성을 부여해 주어 해당 이미지와 컨텐츠의 의미를 반영

<img src=“Images/sample.jpg” alt=“Sample” />

이미지맵으로 구현된 링크

모바일에서는 이미지맵으로 구현된 컨텐츠의 정확한 포지션을 잡지 못해 해당 영역을 클릭해도 액션이 일어나지 않아 사실상 사용이 불가능하다.
이 부분을 해결하기 위해 스크립트 라이브러리가 배포 되고 있으나, 이것을 사용한다하더라도 웹접근성은 확보할 수가 없다. 그러므로 이미지맵을 사용하지 않고 링크요소를 분리하여 모바일용으로 코딩을 해야한다.

  • - IOS : 이미지맵 태그 자체를 인식하지 못해 area태그에 적용된 alt값과 링크 모두 사용이 불가능함.
  • - 안드로이드 : 정상적인 위치로 포커스가 진입하진 않지만, 낭독과 링크의 사용이 가능함. 그러나 area 태그의 alt값과 href 속성을 같이 연결해서 낭독하지 못해, 무슨 링크인지 인지하기는 사실상 어려움.

테이블

caption

CAPTION 엘레멘트를 지정한다면, 그 내용은 그 표의 성격을 설명하여야 한다. CAPTION 엘레멘트는 TABLE의 시작태그 바로 뒤에 만 허용 되며,
한개의 TABLE 엘레멘트에서 하나의 CAPTION 엘레멘트 만을 가질 수 있다.

참고 : http://trio.co.kr/webrefer/html40/struct/tables.html#edef-CAPTION

summary

표의 summary는 요약을 사용자에게 제공한다. 제작자는 표 내용의 요약과, 보이지 않는 사용도구를 사용하는 사람에게 이해를 제공하기 위한 표의 구조를 제공한다.
단, HTML5 문서에서 Table요소에 summary 속성을 이용한 경우, HTML validator 이용시 오류가 발생한다.
이는 HTML5에서는 "align", "bgcolor", "border", "cellpadding", "cellspacing", "frameX", "rules", "summary", "width" 속성들을 더이상 지원하지 않기 때문이다.
W3C에서는 모든사람이 유용한 정보를 얻을 수 있도록 summary 속성으로 제공할 내용을 caption에 포함하거나, table 요소 이전에 화면에 출력되도록 하는 방식으로 제공하는 권장하고 있다.

참고 : http://trio.co.kr/webrefer/html40/struct/tables.html#adef-summary

scope

읽어야 하는 방향 지정. scope="col"의뜻은 세로(column), row는 가로(row)

참고 : http://trio.co.kr/webrefer/html40/struct/tables.html#non-visual-rendering

thead,tbody,tfoot

thead만 있으면 벨리데이터 오류. tfoot과 같은 경우 테이블의 가장 아래에 위치하며, 마크업의 흐름상 thead 다음 tfoot 다음 tbody 순으로 코딩해야함

colgroup

<col>태그를 그룹으로 관리. 행이 아닌 열단위로 스타일 제어가능.
열의 구조적인 그룹화를 위해서 사용하며, <col>을 포함하는 부모요소.
기본값이 '1'이기 때문에 최소 두개 이상의 열을 그룹화할 때에는 <span>속성일 지정하면 됨.

참고 : http://trio.co.kr/webrefer/html40/struct/tables.html#edef-COLGROUP

레이블 제공

입력 서식(<input>, <textarea>, <select> 요소 등)에 대응하는 레이블(<label> 요소, title 속성)을 제공해야 한다.

<!-- for="" 와 id="" 를 연결 해주는 명시적 연결 -->	
					<label for="gender">성별</label>
					<input type="text" id="gender" /> 

					<!-- input type="checkbox", input type="radio" 는 아래와 같이 label 안에 묶는 암묵적 연결 --> 
					<label><input type="radio" name="gender" />여성</label> 
					<label><input type="radio" name="gender" />남성</label>

건너뛰기 링크(Skip Navigation)

건너뛰기 링크는 말 그대로 “링크”를 건너뛰기 위해 제공되는 메뉴를 의미한다. 여기서 “링크”라는 것은 매 페이지마다 반복적으로 제공되는 링크를 말하며, 건너뛰기 링크는 이를 건너뛰어 바로 본문으로 포커스를 이동할 수 있게 해주는 역할을 하게 된다.
반복적인 링크들로 인해 불편해 하지 않고, 원하는 영역 즉 “본문”으로 바로 이동할 수 있는 건너뛰기 링크를 반드시 제공 해야함.

role 속성 적용

ARIA는 특정 요소가 담당하는 역할을 기준으로 role 속성에 지정할 수 있는 몇가지 속성값을 정의한다. 이를 기술적 용어로 온톨로지(ontology)를 부여한다고 표현한다.
페이지 탐색을 목적으로 페이지 내의 특정 지점을 묘사한다. <nav role=“navigation”> 과 같이 사용하며 스크린 리더가 이 영역을 쉽게 이동하도록 만들수 있다.

* HTML5 DTD 적용 서비스에서는 role 속성을 적용함. 이외 문서타입에서는 적용하지 않음 (사용시 validation 오류 발생)

role="banner"：<!-- 특정 문서가 아닌 전체 사이트 영역을 지정, 예를 들어 사이트의 헤더나 로고 -->
					role="contentinfo"： <!-- 메인 콘텐츠의 정보를 지정. (footer의 저작권 정보 표시등) -->
					role="navigation"： <!-- 현재 문서나 관련된 문서의 내비게이션 링크를 지정 -->
					role="search"： <!-- 검색을 수행하는 영역을 정의 -->
					role="complementary"： <!-- 페이지의 메인 섹션을 보완하는 영역을 지정 -->
					role="application"： <!-- 웹 애플리케이션에 사용되는 영역을 지정 -->
					role="form"： <!-- 웹 폼을 지정. 검색 엔진일 경우에는 search를 사용 -->
					role="main"： <!-- 페이지의 메인 콘텐츠를 지정 -->
